Fmax Normalization

Due to the fact that LRE determines target quantity expressed fluorescence units, quantitative accuracy is directly dependent on the optical precision of the qPCR instrument. Extensive work with optical calibration using lambda gDNA as a quantitative standard, has indicated that this is generally not a major concern. However, close examination has revealed that significant well to well differences in fluorescence intensity can occur, at least for some instruments, which can generate quantitative errors in the range of 0.5- to 2.0-fold. While such variance has generally been found to be small, a simple method for quantifying and correcting such variances was developed, based on comparing profile height. Because profile height is directly linked to the fluorescence scale, Fmax can be used to quantify differences between individual wells, in addition to differences between runs. This is expressed as the coefficient of  variation (CV = standard deviation / average X 100%) along with the average Fmax which in turn provides a reference for normalizing the fluorescence units within each individual well within a run, in addition to providing the ability to assess both intra and inter run variance in fluorescent scale.

For example, if a profile generates a Fmax that is 25% higher that the average Fmax, it will generate ~25% overestimation of target quantity. When Fmax normalization is applied, this bias is corrected by simply lowering the predicted target quantity by 25%.

The average Fmax also allows run to run differences in fluorescence scale to be assessed, by comparing their respective average Fmax values. The average Fmax across all runs contained within a database, is presented in the top node within the profile explorer tree panel, along with the corresponding CV. Although these differences are generally small (<10%), this does allow any aberrations in the optical intensity of a run to be easily identified.
